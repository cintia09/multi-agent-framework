CODENOOK · v0.25.5

装在工作目录里的
多智能体任务编排器

一个人写的，目标是把 Claude Code、Copilot CLI 这种命令行 AI 助手从"聊天对话"那一套里拽出来，按工程化的方式干活。

A SINGLE-AUTHOR PROJECT · PYTHON3 · MIT
// 02 — 它解决的问题

session 一关就完了

用过这些助手的都知道，session 一关就完了。今天聊到一半的设计、测试里挖出来的 bug、临时商量好的约定，下一次开窗口全部归零。上下文一压缩，连前半段说过什么都开始记不清，得从头再铺一遍背景。这是基操。

更难受的是受限环境。公司机器、客户机器、跳板机，AI 助手的 GUI 客户端根本装不上。命令行版本能跑，但是没有持久的工作记忆，每次都像新员工第一天上班。

// 03 — 怎么干这件事

一个 .codenook/ 文件夹

CodeNook 的处理方式不复杂。它在你的项目根目录建一个 .codenook/ 文件夹，里面是纯文本，全部是 JSON、YAML、Markdown。没有数据库，没有后台进程，没有要起的服务。需要 Python3，仅此而已。工作空间整个 tar 一打，拖到另一台机器解开就接着用，状态原样保留。

干活的方式从"对话"切到"任务"。你跟 AI 助手说一句"用 codenook 做这件事"，助手就会给你开一个 task，分配 ID 像 T-008-codenook-培训-html-v3 这样。这个任务存在 .codenook/tasks/T-008.../state.json 里，记录当前 phase、status、迭代次数、目标目录、用的模型。session 关了不影响，下次开新窗口，AI 助手自己读这个 state 接着干。

// 04 — 跑起来什么样

phase 一段一段往前推

每个任务按 phase 一段一段往前推。写代码的走 development 插件的流水线，从 clarify 到 design、plan、implement、test、accept、ship；写文章的走 writing 插件，从 outline 到 draft、review、revise、publish；不归类的丢给 generic。三个内置插件都在 .codenook/plugins/ 下面，纯文件，可以改可以加。

phase 之间不全是直通，关键节点上有 HITL gate，就是审批闸口。AI 助手把这一阶段的产出甩给你看，你说 approve、reject 或者 needs_changes，它才往下走。命令是： 

$ codenook decide --task T-008 --phase design --decision approve

不需要审的环节没有 gate，会自动过。

跑起来的模样大概是这样。命令行里 codenook task new --plugin writing --title "培训 HTML"，回车，拿到 task ID。然后 codenook tick --task T-008 --json，每 tick 一次推进一个 phase，吐出 JSON 信封，里面写了下一个该派谁去做、用什么 prompt、产出写到哪个文件。AI 助手按这个信封派 sub-agent 去执行，回来再 tick。循环到 done 或者 blocked 为止。看起来很笨，但是管用。

// 05 — 几个细节

记忆 / 插件 / 迁移

5.1
记忆

抽取 → 索引 → 下次开任务先读

每个 phase 跑完会自动抽一次，把里面有复用价值的东西分三类沉淀下来：knowledge 是工程事实，skills 是可执行的小技能，configs 是配置约定。统一索引到 .codenook/memory/index.yaml，下次开任务时助手会先读这个索引，知道这个仓库之前都干过什么、踩过什么坑。

5.2
插件

不同领域的工作流不互相干扰

development 管软件工程那一摊，writing 管文章那一摊，互不掺和。新插件就是一个目录，放上 phases.yaml、roles/、manifest-templates/，装进去就能用。装的过程也只有一个入口： 

$ python3 install.py --target ~/code/my-project --plugin development --yes

没有 shell 脚本，没有平台分支。

5.3
迁移

顺带的结果

迁移这件事是顺带的结果，不是专门设计出来的。因为所有状态都在 .codenook/ 下面，所有状态都是文本，git 提交可以、tar 打包可以、scp 也可以。换机器、换账号、换助手客户端，活儿不丢。

// 06

它没解决 AI 写代码本身的任何问题。模型该幻觉还是幻觉，该跑题还是跑题。CodeNook 只是把 session 这层临时性的东西，强行钉进了文件系统里。
