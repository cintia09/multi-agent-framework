# Runtime issue investigation example

This example shows the intended shape of an Issuenook seed:

1. collect the user-requested issue context;
2. analyse logs openly without preselecting a root cause;
3. analyse code openly without preselecting a root cause;
4. generate root-cause hypotheses;
5. verify the human-selected hypotheses;
6. conclude with evidence, confidence, and follow-up actions.

Use `seed.json` as a starting point and replace the paths, issue text,
and analysis focus with real data.
